## brisa: 一个现代化的 Go SMTP 接收网关框架

`brisa` 是一个使用 Go 实现的、注重模块化、可扩展性和可观测性的 **SMTP 接收网关框架**。它的核心职责是作为邮件系统的入口，高效、可靠地接收邮件，并通过一个灵活的中间件管道进行过滤和验证

`brisa` 本身不规定邮件的最终去向, 它将最终的处理权（如保存到文件系统、推送到消息队列或直接丢弃）交由用户自定义的中间件来决定。

### 核心设计理念

1.  **关注点分离**: `brisa` 的核心职责是 **提供一个可靠的 SMTP 接收引擎和中间件执行管道**。它将邮件的最终处置（保存、投递等）与接收过程完全分离。
2.  **通过中间件实现最终处置**: 框架不包含任何硬编码的邮件保存或投递逻辑。所有最终动作都应通过在中间件链的末端添加一个或多个处置中间件来完成。
3.  **模块化与可扩展性**: 整个邮件处理流程基于中间件链构建，可以轻松地组合、添加或移除各种验证和处理逻辑。
4.  **可观测性优先**: 从设计之初就集成结构化日志、指标和追踪，确保系统状态透明、易于监控和调试。
